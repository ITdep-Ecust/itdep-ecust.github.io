## 一、注意事项

### 1.1 文件大小限制

请确保上传内容中不包含单个超过100M的文件，否则无法提交。

### 1.2 版权问题

所有华东理工大学校内的资源可以直接引用。但是外部内容请确保无版权问题再引用或者上传。

----

## 二、提交

### 2.1 文件路径规范

#### 网站

本仓库中每个网站项目有一个单独的类别，通常包括网站介绍、维护手册、重新部署手册等，因此上传网站相关内容请遵守如下格式：

- 网站文档根目录下
    - 关于文档: ./README.md
    - 维护: ./maintenance/readme.md
    - 部署: ./deploy/readme.md

!!! info "关于文件"

    如有相关资料补充文件，也放到对应文件夹即可。

#### 其他技术文档

统一放入基础技术文档文件夹即可，暂无分类要求，之后管理员统一处理。

若发现个别目录下未按照上述规范进行文件组织，欢迎提交issues或者PR。

### 2.2 文本书写规范

!!! warning "重要"
    你需要有能力书写MD文件。
	
本仓库中所有文档文件以Markdown进行书写。作为IT人员，请务必熟悉MD语法。（可出教程）

----

### 2.3 PR（Pull Request）

!!! warning "重要"
    你需要有能力使用Git。

首先`Fork`本项目，即通过点击[本仓库](https://github.com/ITdep-Ecust/itdep-ecust.github.io)右上角的`Fork`按键将其克隆至个人账号下得到一个同名仓库，对于该同名仓库你具有完整的读写权限，然后你可以对`Fork`进行任意的修改。你可以将仓库拉取到本地，这样修改更加方便。

修改好个人仓库之后，点击`Contribute`按钮即可创建`Pull Request`，按照Github的指示即可完成贡献，贡献会经过管理人员审核（一般是部长）。
